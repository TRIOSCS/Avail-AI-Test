# Wireframe Screen Spec

## 01_sourcing_results_view.png
Purpose:
- Buyer-facing lead list for a selected part
- Feels like a work queue, not a raw search dump

Key areas:
- Header: part number, run timestamp
- Filters: confidence, safety, freshness, source, status, contactability, corroboration
- Sort: best overall, freshest, safest, easiest to contact, most proven
- Lead cards: vendor, confidence, safety, source badges, freshness, contact preview, suggested next action, quick actions

## 02_lead_detail_view.png
Purpose:
- Explain why a lead exists
- Show all source attribution
- Let buyer act with minimal friction

Key areas:
- Lead summary
- Evidence list
- Source attribution table
- Contact information
- Safety review
- Buyer actions and activity timeline

## 03_buyer_followup_queue.png
Purpose:
- Operational queue for buyers after leads exist

Key areas:
- Status tabs
- Queue table with vendor, part, confidence, safety, last action, next step
- Designed for rapid follow-up and status updates

## 04_safety_review_block.png
Purpose:
- Vendor trust/risk review
- Use caution language and show signals

Key areas:
- Safety band
- Summary
- Positive signals
- Caution signals
- Recommended buyer action
