## 1. Headline

**17 distinct requested-but-unshipped / shipped-but-incomplete items survived verification** (from 19 raw candidates across 7 evidence sweeps; 2 were duplicate pairs and were merged). Split by verdict: **12 confirmed-missing, 5 confirmed-incomplete**. By kind: **11 never-built, 2 built-not-merged, 3 partially-built, 1 stale-claim, 0 merged-but-unreachable**.

The cluster that most deserves a look is **security/launch hardening that reads as "done" but isn't**: the interactive API docs and full OpenAPI schema are served unauthenticated straight through the public edge; the global rate limit is dead code (the middleware was never registered, so only ~41 of the routes throttle); a demoted admin is silently re-promoted on their next login; and the Graph-webhook and password-login "fixes" each shipped only their app-layer/logging half while the prescribed edge allowlist / fail-boot guard were dropped. The rest are genuine feature gaps the user asked for and that fell through the cracks (vendor-facing QP share link, the CRM "honest reporting" page, near-real-time calendar webhooks, the nav-orphaned Buy Plans hub) plus low-stakes cleanup. Every one is either untracked entirely or captured only in a code comment / session-memory handoff, so none will resurface on its own. Zero candidates turned out to be false alarms (no "actually-shipped" or "unclear" verdicts), which is why the "checked and fine" section in §3 is mostly about adjacent facts the verifiers confirmed present.
