# UI and UX Spec

## Primary sourcing interaction model
The sourcing UI should follow a **list → detail** pattern.

### Results View
Each lead card/row should show:
- vendor name
- requested part / matched part
- confidence band
- safety band
- reason summary
- source badges
- freshness
- contact preview
- buyer status
- suggested next action
- caution flags

### Lead Detail View
Must show:
- summary header
- why this lead was generated
- evidence grouped by type
- source attribution
- contact information
- safety review block
- buyer actions
- status/activity history

## Explicit non-features
- no resizable split-pane panels
- no Buyer Follow-Up Queue
- no extra workflow dashboards that create noise

## UI tone
The UI should feel:
- operational
- explainable
- trustworthy
- fast to scan
- cautious but not alarmist
