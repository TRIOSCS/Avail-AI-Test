## CRITICAL Findings (Fix Immediately)

### C1: `db.rollback()` mid-loop wipes ALL prior heals in batch
- **File:** `app/services/integrity_service.py:165-227`
- **Impact:** When `resolve_material_card` raises for one orphan, `db.rollback()` rolls back the entire transaction including all previously healed records. Function continues loop and logs inflated counts.
- **Fix:** Use `db.begin_nested()` (savepoint) per record to isolate failures.

### C2: BUYER/MANAGER blocked from requisitions they didn't create
- **File:** `app/routers/rfq.py:49-55`
- **Impact:** `_enforce_req_scope_for_user` only allows ADMIN bypass. BUYER/MANAGER/TRADER/SALES all get 404 for reqs they didn't create. Contradicts `get_req_for_user` in dependencies.py which only restricts SALES.
- **Fix:** Mirror `get_req_for_user` logic — only filter `created_by` for SALES role.

### C3: Auto-attribution job never commits — all work lost
- **File:** `app/jobs/maintenance_jobs.py:62-82`
- **Impact:** `_job_auto_attribute_activities` calls `run_auto_attribution(db)` but never calls `db.commit()`. Per codebase convention, service functions do NOT commit.
- **Fix:** Add `db.commit()` after the call.

### C4: Redis lock released even when never acquired
- **File:** `app/jobs/core_jobs.py:140-145`
- **Impact:** `finally` block deletes lock key regardless of whether `acquired` was True. When lock acquisition fails, this deletes another worker's lock.
- **Fix:** Only `r.delete(lock_key)` if `acquired` was True.

### C5: 7+ background jobs swallow exceptions silently
- **Files:** `app/jobs/maintenance_jobs.py:127-130,183-186`, `app/jobs/health_jobs.py:60-62,78-80`, `app/jobs/tagging_jobs.py:117,139,161,183,200`
- **Impact:** Jobs catch Exception without re-raising. `_traced_job` decorator and Sentry never fire. Scheduler marks them as succeeded.
- **Fix:** Add `raise` after `logger.exception()` / `db.rollback()` in all affected jobs.

### C6: Token refresh errors silently swallowed
- **File:** `app/jobs/core_jobs.py:138-139`
- **Impact:** When `refresh_user_token` fails, error is logged but not propagated. User's M365 integration silently breaks — RFQs fail to send, inbox scans find nothing.
- **Fix:** Set `user.m365_error_reason = str(e)` and commit, so UI can display warning.

### C7: Inbox scan errors swallowed
- **File:** `app/jobs/core_jobs.py:207-209`
- **Impact:** General Exception handler does NOT set `user.m365_error_reason` (unlike the TimeoutError handler above it). Vendor replies never processed, user has no idea.
- **Fix:** Set `m365_error_reason` in both handlers consistently.

### C8: 4 email sub-operations independently swallow errors
- **File:** `app/jobs/email_jobs.py:407-429`
- **Impact:** `_scan_stock_list_attachments`, `_mine_vendor_contacts`, `_scan_outbound_rfqs`, `_scan_excess_bid_responses` each catch/swallow independently. Parent marks scan "complete" even when sub-ops failed.
- **Fix:** Track sub-op failures and either re-raise or set a partial-failure flag.

### C9: Vendor card creation returns uncommitted object after rollback
- **File:** `app/utils/vendor_helpers.py:155-159`
- **Impact:** After commit fails and rollback, function continues and returns card object with no DB ID. Callers get broken object.
- **Fix:** Return `None` after rollback. Have callers handle None.

## HIGH Findings (Fix Soon)

### Security

#### H1: 3 HTMX admin endpoints missing `require_admin`
- **File:** `app/routers/htmx_views.py:8466,8486,8532`
- **Impact:** `admin_api_health`, `import_vendors_csv`, `admin_data_ops` use `require_user` — any authenticated user can import vendors and view admin dashboards.
- **Fix:** Change dependency to `require_admin`.

#### H2: `|safe` without `|sanitize_html` on email preview
- **File:** `app/templates/htmx/partials/sightings/preview_inquiry.html:59`
- **Impact:** `{{ p.html_body|safe }}` renders HTML without sanitization. If user data flows through `_build_html_body` without escaping, XSS is possible.
- **Fix:** Add `|sanitize_html` before `|safe`.

#### H3: No file size limits on CSV/Excel import
- **Files:** `app/routers/admin/data_ops.py:239,352`, `app/routers/htmx_views.py:8498,505`
- **Impact:** Entire file read into memory with no size check. Memory exhaustion DoS.
- **Fix:** Add `MAX_UPLOAD_BYTES` check before reading.

#### H4: Destructive admin ops via query param
- **File:** `app/routers/admin/data_ops.py:764-799`
- **Impact:** `?dry_run=false` triggers bulk archive/reject/blacklist with no confirmation body.
- **Fix:** Move `dry_run` into request body with explicit `confirm: bool` field.

#### H5: No length caps on user input in Claude prompts
- **Files:** `app/services/ai_service.py:252-262`, `app/services/response_parser.py`
- **Impact:** `user_draft` and `email_body` pasted directly into prompts without length limits.
- **Fix:** Enforce 12,000 char limit on all user-controlled prompt inputs.

#### H6: No file type validation on offer attachment upload
- **File:** `app/routers/crm/offers.py:712-714`
- **Impact:** Any file type (executables, HTML) can be uploaded.
- **Fix:** Add ALLOWED_EXTENSIONS check.

### Data Integrity

#### H7: Cache key omits `domain` parameter
- **File:** `app/services/ai_service.py:189`
- **Impact:** `cache_key = f"intel:{company_name.lower()}"` — different domains for same company get stale cached data.
- **Fix:** Include domain in cache key.

#### H8: AI contacts saved without field-length validation
- **File:** `app/routers/ai.py:167-184`
- **Impact:** Arbitrarily long strings from Claude hit database columns.
- **Fix:** Truncate fields to column max lengths before DB insert.

#### H9: SALES users see all requisition counts
- **File:** `app/routers/requisitions/core.py:67-91`
- **Impact:** `requisition_counts` endpoint has no role filter — SALES users see total system counts.
- **Fix:** Apply same `created_by` filter used in list endpoint.

#### H10: Search refresh returns stale data with no warning
- **File:** `app/routers/sightings.py:515-522`
- **Impact:** When `search_requirement` fails, old data returned as if refresh succeeded.
- **Fix:** Return HTMX toast indicating refresh failed.

#### H11: AI qty estimation falls back to inflated sum
- **File:** `app/services/sighting_aggregation.py:73-75`
- **Impact:** Claude failure → `sum()` of all quantities → same stock counted multiple times.
- **Fix:** Flag the fallback in the response so UI can show "approximate" warning.

#### H12: Credential decryption silently falls back to env var
- **File:** `app/services/credential_service.py:89-93`
- **Impact:** Corrupted encryption → silently uses raw env var (possibly old/wrong credential).
- **Fix:** Add health check flag and admin notification.

## MEDIUM Findings

### Type Design

#### M1: 30 StrEnums defined but 70+ raw string usages
- **Impact:** Typo bugs, no IDE find-all-references.
- **Fix:** Mechanical replacement pass.

#### M2: `list[dict]` in response schemas
- **Impact:** Zero API contract validation. Opaque OpenAPI docs.
- **Fix:** Replace with typed Pydantic models.

#### M3: AI responses untyped
- **Impact:** `claude_structured()` returns `dict | None` — callers have no IDE support.
- **Fix:** Create Pydantic models for AI response schemas.

#### M4: Only 1 `@validates` in 73 models
- **Impact:** No defense-in-depth on model fields.
- **Fix:** Add `@validates` on `User.role`, `Offer.status`, `Requisition.status`, `Sighting.confidence`.

#### M5: No DB CheckConstraints on status columns
- **Impact:** Invalid values can be persisted.
- **Fix:** Add CheckConstraints via Alembic migration.

#### M6: Startup seed functions silently fail
- **File:** `app/startup.py:110-113,141-143,167-169,831-833,865-867`
- **Impact:** App starts but agent auth fails with confusing 503.
- **Fix:** Re-raise on critical seeds (agent user, admin user).

#### M7: Auto-search uses `logger.debug` for failures
- **File:** `app/routers/htmx_views.py:1067`
- **Impact:** Production debug logs typically suppressed — failures invisible.
- **Fix:** Change to `logger.warning`.

#### M8: CSP allows `unsafe-inline` and `unsafe-eval`
- **File:** `app/main.py:309-316`
- **Impact:** Required by Alpine.js but weakens XSS protection.
- **Fix:** Architectural — consider nonce-based CSP long-term.

#### M9: LIKE pattern injection
- **Files:** `app/routers/sightings.py:115,120`, `app/routers/tags.py:32`, `app/routers/crm/offers.py:286`
- **Impact:** Users can inject `%`/`_` wildcards to widen search results.
- **Fix:** Use `escape_like()` on user input.

#### M10: Enrichment connector failures not classified
- **File:** `app/services/enrichment.py:112-114`
- **Impact:** Timeouts, auth errors, rate limits all treated identically — no retry logic.
- **Fix:** Classify errors and retry transients.
