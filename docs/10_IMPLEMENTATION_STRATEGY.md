# Implementation Strategy

## Preferred implementation order
1. Audit current code against this package
2. Repair or complete the lead/evidence model
3. Repair or complete confidence vs safety separation
4. Repair or complete evidence/source attribution rendering
5. Repair or complete buyer status actions
6. Repair or complete safety review presentation
7. Repair or complete suggested next action support
8. Cleanup and QA

## Rule for the coding agent
- audit first
- plan second
- implement one slice at a time
- stop after every phase
- avoid broad rewrites unless clearly necessary
