# React UI Target

This describes the intended React-oriented sourcing UI target, even if the current repo does not match it yet.

## Core screens
1. Sourcing Results View
2. Lead Detail Drawer or Panel
3. Safety Review Block within detail view

## Preferred component model
- `SourcingPage`
- `SourcingFiltersBar`
- `LeadList`
- `LeadCard`
- `LeadDetailDrawer`
- `WhyFoundSection`
- `EvidenceSection`
- `SourceAttributionSection`
- `SafetyReviewSection`
- `BuyerActionsSection`
- `LeadActivityTimeline`

## Important UI rules
- keep confidence and safety visually distinct
- keep source attribution visible
- keep reason summary visible
- keep suggested next action visible
- let buyer update status quickly
- avoid giant all-in-one components
