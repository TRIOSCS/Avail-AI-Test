AvailAI's growth model is **bottom-up adoption within a procurement team** — one buyer activates, sees value, expands to colleagues. The primary growth engineering levers are: activation completeness, habit-forming UX patterns, and compounding value signals (vendor scores, material cards).

## Growth Levers in AvailAI

| Lever | Mechanism | Where to implement |
|-------|-----------|-------------------|
| Activation completeness | User completes first search + RFQ | Empty-state CTAs, onboarding checklist |
| Compounding vendor intelligence | Every search improves vendor scores | Surface score changes in vendor cards |
| AI trust escalation | Confidence ≥0.8 auto-applies; visible to user | Confidence microcopy in offer review |
| Team expansion | Share requisition or buy plan with colleague | Requisition detail share CTA |

## Activation Loop Patterns

The fastest path to retained users is getting them through the full search-to-RFQ loop on day one.

```html
{# app/templates/htmx/partials/requisitions/detail.html — guide to next step #}
{% if requirement_count > 0 and not any_rfq_sent %}
<div class="rounded-md bg-blue-50 border border-blue-100 px-4 py-3 text-sm text-blue-700">
  <span class="font-medium">Search complete.</span>
  Select vendors from the results below and send your first RFQ.
</div>
{% endif %}
```

### Progress Indicators

```html
{# Inline progress — shows user where they are in the workflow #}
```

## Retention Hooks in Templates

### Compounding Intelligence Signal

Every search adds to vendor reliability scores. Surfacing this creates a retention hook: "leaving AvailAI means losing your vendor intelligence."

```html
{# app/templates/htmx/partials/vendors/vendor_card.html — surface score history #}
<div class="text-xs text-gray-500 mt-1">
  Reliability score: <span class="font-medium text-gray-900">{{ vendor.reliability_score }}/100</span>
  {% if vendor.score_trend > 0 %}
    <span class="text-green-600 ml-1">+{{ vendor.score_trend }} this month</span>
  {% endif %}
</div>
```

### Proactive Match Count Badge

```html
{# Nav badge — shows pending proactive matches to drive re-engagement #}
<a hx-get="/v2/proactive" hx-target="#main-content" class="flex items-center gap-2">
  Proactive Matching
  {% if proactive_count > 0 %}
  <span class="ml-auto inline-flex items-center px-1.5 py-0.5 rounded-full
               text-xs font-medium bg-brand-100 text-brand-700">
    {{ proactive_count }}
  </span>
  {% endif %}
</a>
```

## Referral and Network Effects

AvailAI is a single-tenant B2B app — there is no viral loop with external users. Growth is intra-team: a buyer uses it daily, a manager asks about it, more seats are added.

**Surface value to non-users via exported artifacts:**

```html
{# app/templates/documents/quote_report.html — PDF footer #}
<footer class="text-xs text-gray-400 text-center mt-8">
  Generated by AvailAI — Trio Supply Chain Solutions
</footer>
```

Quote PDFs sent to customers carry the AvailAI brand. This is the primary referral surface — keep it clean and professional.

## WARNING: Growth Hacks That Erode Trust

**The Problem:**

```html
{# BAD — artificial urgency on a B2B tool #}
<p class="text-red-500 text-xs animate-pulse">
  Act fast — 3 vendors have this part in stock!
</p>
```

**Why This Breaks:** Procurement buyers make deliberate purchasing decisions. Fake scarcity patterns are immediately recognized and destroy trust in the platform's data integrity — which is AvailAI's core value proposition.

**The Fix:** Only surface real, data-backed signals.

```html
{# GOOD — real signal from search result #}
{% if sighting.quantity_available < target_qty %}
<span class="text-xs text-yellow-600">
  Qty available ({{ sighting.quantity_available }}) is below your target ({{ target_qty }}).
</span>
{% endif %}
```
